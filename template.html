Introduction

Everything you need to know to get up and running quickly.


Tip: you can toggle between light and dark mode using the button in the top-right corner.

Installation

Install via your package manager of choice:

npm install your-package-name

Or using yarn:

yarn add your-package-name

Quick Start

Here's a minimal example to get something running on screen:

import { init } from "your-package-name";

init({
  target: "#app",
  theme: "auto"
});

That's it — your app should now be initialized and ready to use.

Configuration

Configuration options can be passed when initializing the app, or defined in a config file at the root of your project.

OptionTypeDefaultDescription
themestring"light"Sets the default theme.
strictbooleanfalseEnables strict mode checks.
localestring"en"Default locale for content.

Routing

Routes are defined declaratively and map paths to views. Nested routes are supported out of the box.

routes: [
  { path: "/", component: Home },
  { path: "/docs/:slug", component: DocPage }
]

Components

Components are reusable, self-contained building blocks. They can accept props, emit events, and be composed together.

  • Stateless presentational components
  • Composable layout primitives
  • Lifecycle-aware containers

Endpoints

GET /api/v1/items

Returns a paginated list of items.

POST /api/v1/items

Creates a new item.

Authentication

All requests must include a bearer token in the Authorization header:

Authorization: Bearer YOUR_API_KEY
